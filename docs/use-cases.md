# Use Cases (варианты использования)

## UC-01 Регистрация пользователя
**Актор:** Гость
- Предусловия: нет
- Основной сценарий:
  1) Открыть `/register/`
  2) Ввести username/email/пароль
  3) Получить учётную запись с ролью «Инженер» и попасть на дашборд
- Результат: пользователь создан, авторизован

## UC-02 Вход в систему
**Актор:** Пользователь
- 1) Открыть `/login/`
- 2) Ввести логин/пароль
- 3) Попасть на дашборд

## UC-03 Создание дефекта
**Актор:** Инженер или Менеджер
- 1) Нажать «Создать дефект»
- 2) Заполнить поля (проект, заголовок, описание, приоритет, deadline)
- 3) Сохранить
- Результат: дефект создан, записана история события

## UC-04 Назначение исполнителя
**Актор:** Менеджер
- 1) Открыть дефект
- 2) «Редактировать» → выбрать исполнителя
- 3) Сохранить
- Результат: исполнитель изменён, история обновлена

## UC-05 Смена статуса по workflow
**Актор:** Исполнитель (инженер) / Менеджер
- 1) Открыть дефект
- 2) Нажать кнопку перехода статуса
- Результат: статус изменён согласно workflow и роли, история записана

## UC-06 Комментарии и вложения
**Актор:** Пользователь с доступом к дефекту
- 1) Добавить комментарий / загрузить файл
- Результат: запись добавлена, история записана

## UC-07 Экспорт отчётов
**Актор:** Менеджер/Руководитель/Инженер
- 1) На дашборде нажать «Экспорт CSV/Excel»
- 2) Скачать файл
- Ограничение: инженер видит только свои дефекты

## UC-08 Аналитика
**Актор:** Менеджер/Руководитель
- 1) Открыть `/analytics/`
- 2) Посмотреть график и таблицу по статусам
